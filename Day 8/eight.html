Day 8 Learning Document Object Model (DOM)

The Document Object Model (DOM) is a programming interface for web documents, specifically HTML and XML. It represents the structure of a webpage as a tree of objects, where each object corresponds to a part of the page, such as elements (e.g., div, p), attributes (e.g., id, class), and text content.

In JavaScript, the DOM allows for programmatic interaction with web documents. It provides methods and properties to: Access elements: Select specific HTML elements based on their tag name, class, ID, or other attributes using methods like document.getElementById(), document.querySelector(), and document.querySelectorAll(). Modify content: Change the text content (textContent), inner HTML (innerHTML), or attribute values of elements. Change style: Dynamically apply or modify CSS styles to elements using the style property. Add and remove elements: Create new HTML elements, append them to the document, or remove existing elements from the document. Handle events: Respond to user interactions such as clicks, form submissions, key presses, or mouse movements by attaching event listeners to elements.

Everything in the DOM tree is a node, and nodes can represent different parts of the document:

  • Comment Nodes: Represent HTML comments within the document.
  • Element Nodes: Represent HTML tags like p, div, and h1.
  • Text Nodes: Represent the text content within an element.
